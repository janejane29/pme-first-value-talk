Benchmark

이미 잘 풀고 있는 제품들의 공통점

설정이 아니라 목적부터 묻고, 빈 화면 대신 시작점을 주고, 전문 지식을 시스템에 맡긴다.
Goal-first
“무엇을 하려는지”부터 묻는 진입점. 목적에 맞는 것만 보여준다.
Template-based
빈 캔버스가 아니라, 좋은 시작점을 미리 제공한다.
Layered
복잡한 설정은 나중 레이어로 분리해 단계적으로.
Insight

“빈 화면 대신 시작점을 주고,
전문 지식을 시스템에 묻고, 결정의 수를 줄였다.”

관점 1시작 장벽 감소 — “이걸 어떻게 시작하지?” 대신 “이걸 조금 고치면 되겠네”
관점 2도메인 지식 대체 — 전문지식 없이도 결과물을 만들 수 있다
관점 3의사결정 축소 — 내려야 할 결정이 줄고, 속도가 빨라진다
관점 4학습 가이드 — 쓰면서 도메인 구조를 자연스럽게 학습한다
Direction

“파라미터”가 뭔지도 모르는 유저가
막힘없이 원하는 캠페인을 운영할 수 있을까?

Insight

에어브릿지에 녹일 수 있는 3가지

1
Intent-first
사용자에게 입력을 받지 말고, 의도를 묻는다.
2
System-as-Translator
도메인 지식을 사용자에게 학습시키지 말고, 시스템 안에 담는다.
3
Progressive
처음엔 단순하게, 복잡한 건 필요할 때만 펼친다.
Flow

사용자는 의도만 던지고,
Pilot이 21개를 알아서 채운다

Flow 1사용자 의도 입력 — “인스타에 여름 세일 캠페인 링크 만들고 싶어요”
Flow 2맥락 분석 — 이전 캠페인 · 채널 특성 · 시즌 패턴을 Pilot이 파악
Flow 3유사 캠페인 매칭 — “작년 7월 운영하셨던 것처럼 만들까요?”
Flow 4링크 자동 생성 — 사용자는 의도만, Pilot이 21개 필드를 채운다
* Pilot은 에어브릿지 제품에 내장된 자체 AI 에이전트입니다.
Ideation

이미 운영하던 패턴이 있다면,
Pilot이 먼저 제안할 수도 있다

“작년 5월 인스타 캠페인이 효율이 좋았어요.
이번에도 비슷하게 가시겠어요?”
Value
우리는 그동안 유저를 학습시키려고 했다.

누구나 앱을 만드는 AI 시대에,
처음 쓰는 사용자가 빠르게 First Value를 느낄 수 있도록
우리의 사고를 전환하자.

이건 트래킹 링크 하나의 이야기가 아니다.
모든 티켓, 모든 스쿼드가
같은 Goal로 정렬되어야 한다.

“여러분의 티켓에서,
이 관점은 어떻게 적용될 수 있을까요?”
